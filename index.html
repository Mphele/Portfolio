things built

name what it does stack year
doria security scanner for AI-assisted dev python · rust · js 2025 →
Catches slopsquatting attacks — when an LLM hallucinates a package name that doesn't exist, an attacker can register it with malware. Doria scans import metadata, cross-references real registries, and flags hallucinated package names before they reach disk. No code execution. The scanning engine is Rust for speed; the CLI and plugin layer are Python; the dashboard is vanilla JS. Won national champions at the ITWeb Security Summit Hackathon 2026 out of every competing team in the country.
fig 1. doria cli flagging hallucinated packages
uwatu event-driven agritech platform go · typescript · python 2024 →
Bridges IoT telemetry from livestock sensors, telecom network signals, and ML models to monitor animal health in real-time. Go backend handles event ingestion and routing at scale. Python layer runs predictive models that flag early signs of illness. TypeScript frontend visualizes herd status. Built for African agricultural contexts — intermittent connectivity, high stakes, low margin for error, and animals that can't wait for a retry.
fig 2. uwatu telemetry simulator
afyaai health triage assistant python · fastapi 2024 →
Conversational healthcare access powered by Gemini. Triages symptoms, suggests next steps, and knows when to escalate to a human. FastAPI backend with streaming responses so the user isn't staring at a spinner. Designed for communities where seeing a doctor isn't always an option — the API is deliberately simple because the users can't afford complexity. Won 1st place at the Mjanga AI Hackathon, Africa-wide.
fig 3. fastapi auto-docs for triage endpoints
sieve recruiter-first ATS python · fastapi 2024 →
An ATS built to put recruiters back in control. Instead of replacing human judgment, Sieve surfaces what matters and flags false positives from automated screening. AI-assisted ranking that respects recruiter intuition over algorithmic authority. Python/FastAPI backend with a clean REST API. The core insight: most ATS systems optimize for the hiring manager. Sieve optimizes for the person who actually reads the resumes.

what i reach for

Day to day I write Java with Spring Boot, Python, and Go — Java for robust, enterprise-grade services, Python for data pipelines and ML integration, Go for concurrent, high-performance systems. I've also shipped code in Rust and TypeScript when the constraints demanded it. Bash for everything in between.

For state, PostgreSQL is my default — I'll reach for MySQL or SQLite when the constraints call for them. FastAPI for Python services because the async story is clean and the OpenAPI docs come for free. Linux on every machine I own. Docker for anything that needs to run somewhere else, and I'm working toward Kubernetes for the things that need to run everywhere.

I'm not religious about tools. I'm religious about APIs — because a well-designed API is the foundation of any great user experience, and a badly designed one is a tax every future developer pays. I'll learn your stack. I'll have opinions about it. I'll be honest about them.
